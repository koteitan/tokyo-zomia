# 構成
- ダウンロードアプリ: APIからのデータのダウンロードと表示用の解析を行う
- データ: ダウンロード・解析されたデータ
- web アプリ: 表示用のアプリ。
データとweb アプリは公開されます。

# ダウンロードアプリ
- data/*
- linux/WSL2 で実行
- 実行すると data/ に下記のデータがダウンロード・解析される
  - 対象源流河川(後述)のポリラインデータ
  - 対象範囲(後述)の海岸線
## 制約
- 実行中は標準出力に進捗が表示されること
- 対象源流河川以外のデータをあまり無駄にダウンロードしないような工夫がなされていること
- アクセス過多にならないよう、データベースの決まりを守ってアクセス頻度をコントロールすること

# 用語
- 源流: 源流とは、河川の流れ始める点のことである。
- 河口: 河口とは、河川が流れ終わる点のことである。
- 分岐: 分岐とは、河川が複数に分かれる点のことである。
- 合流: 合流とは、複数の河川が合流する点のことである。
- 指定範囲:指定範囲とは、神奈川県早川河口以東、かつ利根川河口以南、のbbox に入る範囲のことである。
## 対象源流河川
対象源流河川とは、下記の条件を満たすポリラインのことである:
- 標高300m以上の源流から、指定範囲の河口に繋がるまでの経路に存在するあらゆる河川のポリライン。

# チーム構成
agent team によって分担する。
## 役割
### planner
- programmer, tester に指示を出す。
- レポートファイル以外のファイル変更は行わない。
### programmer
- 実装を行う。
### tester
- テスト設計、テスト実施を行い、planner に結果を報告する。
- レポートファイル以外のファイル変更は行わない。
## フロー
- planner が下記を満たすプランを立てる。
  - planner が仕様を作成する。
  - tester がテスト仕様を作成する。
  - planner が確認、必要なら修正指示。
  - user が確認。
  - programmer が実装する。
  - programmer がホワイトテストを設計して、テスト実施する。合格すれば次に進む。
  - tester がテスト実施し、planner に結果を報告する。
  - 最初に戻る
## 成果物の保全
- programmer のホワイトテストスクリプトは test/white/ に残すこと。
- tester のテストスクリプトは test/blackbox/ に残すこと。
## テストの分離
- tester は test/white/ のホワイトテストコードを参照してはならない。
- tester は仕様書（docs/）のみを基にブラックボックステストを設計・実施すること。
## チームの解散
- user の指示がない限りチームを解散しないこと。
